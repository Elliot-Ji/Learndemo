适配器模式: 属于结构型模式
          将某个类的接口转换成客户端期望的另一个接口表示,主要目的是兼容性,让原本因接口不匹配
          不能一起工作的两个类可以协同工作.其别名为包装器(Wrapper)
          主要分为三类: 类适配器模式, Adapter类 extends 源类
                      对象适配器模式, Adapter类 持有源类的实例(聚合),
                      接口适配器模式,适用于一个接口不想使用其所有的方法的情况
          作用: 将一个类的接口转换成另一个接口;
                用户调用适配器转化出来的目标接口方法,适配器再调用被适配器的相关接口方法;

          应用:
               springMVC中的HandlerAdapter使用了适配器模式,即请求过来时,会调用不同的适配器(调用不同的controller)进行处理不同的请求,
               而不是直接调用controller;
